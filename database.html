Overview

kalima-db is a SQLite database containing the Quranic text with full morphological annotation. It serves as the foundation for all analysis performed by kalima-mcp and kalima-math.

Location: HuggingFace: Syntra-Env/kalima-db

The database is automatically downloaded on first use by kalima-mcp, or can be manually downloaded using the HuggingFace Hub API.

Data Structure

The database contains:

  • Text content: The Quranic text (Arabic and normalized)
  • Morphological features: Root, POS, verb form, aspect, person, number, gender, case
  • Linguistic annotations: Word types, word instances, morpheme types
  • Reference data: Traditional tafsir translations (Sahih International)

Storage & Distribution

The database is:

  • Stored as kalima.db on HuggingFace Datasets
  • Downloaded via huggingface_hub on first run
  • Cached locally at ~/.scholar/data/kalima.db

This approach keeps the repository lightweight (no large binary files in git) while ensuring the data is versioned and accessible.

Access

Access via kalima-mcp tools, or directly through the database layer in kalima-mcp:

from src.db import get_connection
conn = get_connection()

Manual download:

from huggingface_hub import hf_hub_download
db_path = hf_hub_download(repo_id="Syntra-Env/kalima-db", filename="kalima.db", repo_type="dataset")

Open Questions

The database contains 232 open research questions, 469 hypotheses, and 139 entries in active verification. These drive the ongoing research:

NCU Framework

  • How can characters exist in separate NCUs yet have coherent narratives?
  • Are NCUs "meeting places" in afterlife dimensions?
  • What is the relationship between individual nafs and collective groups in the afterlife?

Mathematical Structure

  • Can Quranic text be represented as a Calabi-Yau manifold?
  • Do semantic shifts correspond to mathematical phase transitions?
  • What is the connection between topological features and semantic content?

Linguistic Analysis

  • Do long vs short alif distinctions reliably indicate this-life vs afterlife orientation?
  • What is the optimal schema for storing root vector profiles?
  • How to ensure content_addresses are unique and efficiently queriable?

Methodology

  • What specific criteria distinguish "validated" from "actively verifying" status?
  • How many contradictory instances invalidate a hypothesis?
  • How should we weight distant references vs. proximate context?

Database Engineering

  • What indexes are needed for common query patterns?
  • Should we precompute common aggregations?
  • How to optimize for both text search and structural queries?
  • Should we add more tafsir sources beyond Sahih International?

These questions evolve as research progresses. New questions are added, some move to hypothesis, and verified findings are locked in.
